Not just an AI chatbot. A system where AI personas can access a real terminal, query local models, look things up, and bring back information — extending their intelligence in real time.

What It Is

Beyond Chat — AI with a Terminal

The Advanced Interface

  • React/TypeScript app — runs in browser, fully offline-capable
  • Characters have their own terminal window
  • Characters can query aichat (--serve mode) for research
  • Characters can query local ollama models directly
  • Plugin system for Image, TTS, and Terminal configuration
  • Per-character JavaScript hooks run before each response
  • Command approval gates: once / this session / always

Why the Terminal Matters

An AI character that can only answer from its training data is limited. A character that can open a terminal, ask a local Qwen model a specialized question, pull an API status, or check a config — that character has real agency.

The terminal is not a gimmick. It is the extension cord that connects the character's personality to the infrastructure it lives on.

Architecture

Character → Terminal → AI → Response

Click any step to understand it.

1
Character Definition
Persona, system prompt, service (AI Horde / Ollama / Gemini / etc.), model, API key. All configurable per-character.
2
Pre-Response JS Hook
Each character has a personal code box. JavaScript runs before each response. Can modify context, inject data, trigger terminal commands.
3
Terminal Command — Approval Gate
Character issues a terminal command. User sees it: approve once / this session / always. Nothing runs silently. Full transparency.
4
aichat / ollama Query
Terminal sends the query to the configured endpoint. aichat (--serve :3030) or ollama (:11434). URL configurable in Plugin Manager, not hardcoded.
5
Response Returned to Character
Terminal output is returned to the character's context. The character can incorporate the research into its reply. Real-time knowledge augmentation.
Plugin System

Configurable from the UI — Nothing Hardcoded

// Three Core Plugins — Global System Configuration

All AI endpoint URLs, image generators, and TTS engines are configured through the Plugin Manager. No editing config files. No hardcoded URLs. Change the aichat port in the UI, it updates everywhere instantly.

default-terminal

aichat base URL + ollama base URL. Used by all characters' terminal sessions. Protected — can't be deleted or disabled.

CORE

default-image-generator

AI Horde image endpoint. Characters can request images. Free via AI Horde, configurable for Stability AI or FLUX.

CORE

default-tts-narrator

Text-to-speech endpoint. Characters can speak. Flite preferred (local), configurable for cloud TTS providers.

CORE

Additionally, each character can have custom plugins — sandboxed JS that runs in their context, enabling per-character logic, memory hooks, or API calls.
